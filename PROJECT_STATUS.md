## 本轮已完成的 P0

### 真实仪表与安全控制

- Real CMW500 支持 VISA 与 SCPI Socket，连接时校验 R&S CMW 身份。
- 真实模式彻底关闭模拟回退；通信、超时、解析和 BLER 非法值全部 fail-closed。
- SCPI 命令区分 `write`、`query`、`query_and_assert`，查询响应不会遗留在传输缓冲区。
- Attach/测量状态使用精确匹配，避免 `CONN` 误匹配 `DISCONN`、`RDY` 误匹配 `NOTRDY`。
- 结构化记录每条 SCPI 的阶段、命令、响应、成功状态、错误和 UTC 时间。
- Stop 已幂等；支持协作取消、I/O 中断、断线重连后 Cell OFF/Cleanup。
- 正常停止终态为 `STOPPED`；无法确认安全清理时锁存 `FAILED_UNSAFE`，后续状态不能覆盖，在操作员明确确认人工 RF/Cell OFF 前禁止新测试和退出。
- 活动测试关闭流程已消除 UI 线程与工作线程互等死锁。
- 后台连接/ADB 操作与测试互斥，不会在运行中替换仪表引用或错配 trace。

### 扫描、线损与结果判定

- 按 `max_step` 粗扫，在首个确认 FAIL 区间按 `min_step` 细扫。
- FAIL 和测量异常均按 `retry_count` 重试，每次原始尝试均保留。
- BLER 必须是有限值且位于 `0..100`；门限同样进行范围校验。
- DUT 目标电平 + 全局线损 + Excel 信道线损形成仪表下发电平。
- BW、包数和总线损传入仪表准备阶段并写入结果。
- 汇总按 Run ID 隔离，以最终重试结果计算灵敏度，并依据规格上限判定。

### 可追溯报告

- 每次测试生成唯一 Run ID，开始时清空上一任务，结束后自动保存独立报告。
- 元数据包含数据来源、仪表 IDN、实际连接参数、DUT/操作人员、APK 哈希、配置路径与 SHA-256、校准信息、软件版本、构建 commit 和配置快照。
- Excel 包含 `RawResults`、`Summary`、`RunMetadata`、`SCPITrace`。
- Fake 结果明确标记 `SIMULATION`，并在 UI 与 Excel 中显示红色水印。
- `STOPPED`、`FAILED`、`FAILED_UNSAFE` 报告显示不完整/不安全水印，不能被识别为正式 PASS。
- Raw/Summary/Metadata/Trace 外部字符串均防止 Excel 公式注入。
- 日志、报告、截图和用户配置迁移到 `%LOCALAPPDATA%\cmw500_auto_test`。

### 桌面端与发布工程

- 耗时的仪表连接、IDN、VISA 扫描和 ADB 操作在后台线程执行。
- 测试期间锁定会改变仪表、模板、信道或 DUT 的控件；关闭窗口先执行安全停止。
- 当前商业候选版只开放 LTE，WiFi/WCDMA/GSM 占位页已禁用。
- 运行依赖与构建依赖已精确锁定，并加入 `pyvisa-py` 内置后端。
- 构建强制复制并验证 config/configs、版本、构建身份和文档等运行资源。
- 归档使用严格版本名、内容校验和 SHA-256 清单。
- GitHub Actions 已拆分 test/build/release 最小权限，并固定 Actions commit SHA。
- 正式版本构建与归档均强制 Authenticode 签名验证；无证书或签名无效时停止发布。

## 商业发布仍未完成的外部门禁

以下事项需要真实硬件、组织证书或业务/法务输入，不能仅靠代码完成：

1. 明确目标 CMW500 型号、固件/软件版本、已安装选件、连接方式和 LTE 应用模式。
2. 依据官方手册和实机响应校准推荐 SCPI 模板，确认 Cell、Attach、RSEPRE 与 EBLER 的全部命令及返回字段。
3. 使用真实 DUT 完成至少一组代表性 Band/信道/电平端到端测试；归档 Excel、日志、SCPI trace 和人工对照结果。
4. 确认仪表校准证书、有效期、线损文件来源、环境条件和复测周期。
5. 在 GitHub 环境配置组织 Authenticode 证书及时间戳服务，并对最终安装包执行签名验收。
6. 完成第三方依赖许可证、隐私说明、EULA/用户手册、客户支持与缺陷响应流程评审。

上述 1–6 未全部签字通过前，不得将当前构建标记为正式量产商业版。
