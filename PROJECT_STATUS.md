# 项目当前状态

最后更新：2026-07-15  
当前阶段：Phase 8 —— LTE 自动化测试原型闭环

## 项目目标

本项目是面向 Rohde & Schwarz CMW500 的手机灵敏度自动化测试桌面工具。当前重点是建立可配置、可观察、可导出结果的 LTE 测试流程，并逐步接入真实仪表和真实终端。

## 已完成

### LTE 测试流程与界面

- 三栏桌面界面：配置与操作区、实时结果区、实时日志区。
- LTE 配置、Band 与信道选择、测试计划生成。
- 后台测试执行，支持开始、暂停、继续、停止，并可安全恢复界面按钮状态。
- LTE 状态机：小区配置 → 开小区 → 等待 UE Attach → 测量 → 清理。
- 相同 Band/信道下扫不同接收电平；切换 Band/信道时重建小区。

### 仪表与测试数据

- Fake CMW500 可完整模拟 LTE 测量流程，用于日常开发、演示和流程验证。
- Real CMW500 支持 TCP Socket/SCPI 连接、断开、`*IDN?` 查询、`*RST` 和 `SYST:PRES`。
- 支持通过 YAML/JSON 加载 LTE SCPI 模板，覆盖 setup、Cell ON/OFF、等待 Attach、设置 RX Level、读取 BLER 和 cleanup 等步骤。
- 已提供 CMW500 推荐 LTE 模板，包含 RRC Connected Attach 检测，以及 EBLER 的启动、完成状态轮询、停止和结果读取流程；仍须经实机校准后使用。
- 支持测试结果判定、灵敏度点汇总和 Excel 导出（`RawResults`、`Summary` 两个工作表）。

### 配置与终端辅助能力

- 可加载 LTE 信道配置 Excel，并优先使用其中的 Band/频点/信道映射。
- 可加载 YAML/JSON 串口配置文件。
- 提供基础 ADB 操作：刷新设备、安装 App、重启、启动/停止 App、清除数据、截图。
- 已提供 Windows 打包脚本和 GitHub Actions 构建/Release 流程。

## 当前限制与已知缺口

- Real CMW500 的 LTE SCPI 指令仍是可替换模板/示例，尚未根据实际仪表选件、应用模式和测试场景完成实机校准。
- 尚未实现真实 pyvisa 连接，也未实现串口的实际收发与设备控制；当前串口功能仅解析配置。
- WiFi、WCDMA、GSM 目前只有配置界面占位，尚未具备测试计划、仪表控制、判定和导出闭环。
- 未实现数据库持久化、历史测试记录管理和正式报告生成。
- 自动化测试覆盖有限：目前基础测试通过，但状态机、SCPI 模板、异常恢复、ADB、导出等关键路径仍需补充测试。

## 最近验证结果

- `python -m pytest -q --basetemp=.pytest-tmp`：15 项通过。
- 当前 Windows 环境的默认 pytest 临时目录存在访问权限问题；运行测试时应指定仓库内的 `--basetemp`，直至该环境问题被修复。
- 当前存在 6 条 pytest 收集警告：部分业务类以 `Test` 开头而被 pytest 尝试收集；不影响测试结果，但应在后续清理。
- GitHub Actions 已配置 Windows 构建与按 `v*` 标签发布。

## 下一步计划

### P0：完成真实 LTE 测试闭环

1. 确认目标 CMW500 的型号、软件版本、已安装选件、连接方式和 LTE 应用模式。
2. 依据仪表手册与实机返回值，校准 LTE 小区配置、开关小区、UE Attach、RX Level 和 BLER 的 SCPI 模板。
3. 在真实手机上完成至少一个 Band/信道/电平扫描的端到端验证，并保存可复用的模板与测试配置。
4. 完善超时、SCPI 错误查询、断线重连、失败清理和可诊断日志。

### P1：提升可用性和质量

1. 为配置文件、信道表和 SCPI 模板增加启动前校验与明确错误提示。
2. 扩充单元/集成测试，优先覆盖状态机、模板变量渲染与解析、异常清理、结果汇总和 Excel 导出。
3. 清理 pytest 收集警告。
4. 实现真实串口通信；如实际仪表环境需要，再补充 pyvisa 传输层。

### P2：扩展制式与数据管理

1. 按 LTE 的成熟流程扩展 WiFi、WCDMA、GSM 的配置、计划、仪表控制、判定和导出。
2. 引入本地数据存储，保存测试任务、原始结果、汇总结果、日志和设备/仪表信息。
3. 生成更完整的测试报告，并支持历史记录查询与对比。

## 下次继续时的建议

优先从 P0 开始：先拿到真实 CMW500 的型号/选件信息、当前可用的手工 SCPI 命令或仪表操作步骤，再将其固化到 LTE 模板并进行小范围实测。未完成实机闭环前，WiFi/WCDMA/GSM 的扩展价值较低。
